How it works

Three steps, browser to CAD.

01

Generate in the browser.

Use the Text-to-CAD web app to describe what you need in plain English. Designmatix returns a CadQuery Python script plus a JSON parameter manifest — the .dxbundle. Save it anywhere on your machine.

02

Open in Companion.

Drop the bundle into the desktop app. Companion reads the manifest and renders an editable form — one input for every variable in the script. Change the tooth count, the bore diameter, the keyway width. Hit regenerate and the STEP file rebuilds locally.

03

Inspect and iterate.

Open the regenerated STEP in your CAD tool of choice, or pop the underlying Python into CQ-editor to see exactly what built the geometry. Tweak the script directly if you want — it's plain CadQuery, readable and runnable.

One script. Many variations.

The same bundle, with different parameter values, produces a different part — no re-prompting, no new generation, no token spend. Edit the form, hit regenerate, you get a new STEP.

Why a desktop app

Real artifacts. Yours to keep.

The Python source is yours.

Every bundle includes the CadQuery script that built the model. Read it, run it, modify it, version-control it alongside the rest of your project. The script is the recipe — and the recipe is in your hands, not behind a vendor's API.

No mesh-only black box.

The output is a real STEP file (ISO 10303) — parametric solid geometry, not a triangle soup. Open it in SolidWorks, Fusion, FreeCAD, NX, Catia. Add it to assemblies. Send it to manufacturing. It's a first-class CAD artifact, indistinguishable from one a human modeller would produce.

Offline regeneration, by default.

Once you have the bundle, Companion regenerates the STEP locally on your machine. No cloud round-trips for each parameter change. No telemetry. Your geometry and your iteration history stay where you put them.

A bundle is a versionable file.

The .dxbundle is a real artifact — script plus parameters, plain text, diff-able. Commit it next to your drawings. Branch it. Review it in a PR. It's not a vendor blob, it's a file.
